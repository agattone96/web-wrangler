# Contributing to WebWrangler

Thank you for your interest in helping with WebWrangler! We're excited to have you here.

## 🌈 Our Vision

WebWrangler is a premium, privacy-focused productivity tool. We value **performance**, **aesthetic excellence**, and **user privacy** above all else. When contributing, keep these core values in mind.

## 🛠 Getting Started

1.  **Fork the Repository**: Create a personal fork of the project on GitHub.
2.  **Clone the Repository**: 
    ```bash
    git clone https://github.com/your-username/web-wrangler.git
    ```
3.  **Install Dependencies**:
    ```bash
    npm install
    ```
4.  **Create a Branch**: Use a descriptive branch name (e.g., `feature/improved-neon-glows`).

## 🎨 Design Guidelines

We use a "Neon Cyber-Goth" design system. If you are adding new UI components:
- Use **Vanilla CSS** instead of utility frameworks.
- Prioritize **Glassmorphism** and high-contrast neon accents.
- Ensure all interactive elements have subtle micro-animations.

## 🔒 Security Practices

WebWrangler blocks ads and trackers by default.
- Never add external scripts from untrusted CDNs.
- Localize all libraries whenever possible.
- Adhere to Electron security best practices (`contextIsolation: true`, `nodeIntegration: false`).

## 🐞 Reporting Bugs

Please use the GitHub Issue tracker to report bugs. Include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Your OS and Node.js version.
- Screen captures (if applicable).

## 🚀 Submitting Pull Requests

1.  Keep PRs focused on a single change.
2.  Update documentation if you're adding new features.
3.  Ensure your code follows the existing style and linting rules.
